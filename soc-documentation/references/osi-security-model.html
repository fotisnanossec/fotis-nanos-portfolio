OSI Model - SOC Analyst Security Reference

Layer-by-layer threat analysis and tool mapping for systematic network security investigation

SOC Analyst Application

This OSI security model demonstrates systematic network analysis capabilities essential for SOC operations, integrating technical knowledge with practical investigation workflows. Each layer provides specific threat vectors, monitoring tools, and investigation techniques for comprehensive incident response.

7

Layer 7 - Application

Security Focus: Application vulnerabilities, malware, user behavior

Common Threats

  • • Web application attacks (XSS, SQL injection, CSRF)
  • • Email-based threats (phishing, malware attachments)
  • • Application-layer DDoS
  • • Credential harvesting
  • • Social engineering

Indicators to Monitor

  • • Unusual HTTP methods or response codes
  • • Large file uploads/downloads
  • • Failed authentication attempts
  • • SQL error messages in logs
  • • Suspicious user-agent strings

SOC Analyst Tools & Techniques

Web Traffic Analysis

Burp Suite, Browser dev tools, Log analysis

curl -I suspicious-website.com wget --spider -S suspicious-website.com

Email Security

Email security gateways, Attachment sandboxing, SPF/DKIM/DMARC validation

6

Layer 6 - Presentation

Security Focus: Encryption/decryption, data formatting, compression

Common Threats

  • • SSL/TLS vulnerabilities
  • • Certificate spoofing
  • • Encryption downgrade attacks
  • • Data format exploits
  • • Compression-based attacks

Investigation Focus

  • • Certificate validity and chain of trust
  • • Cipher suite strength assessment
  • • Protocol version analysis
  • • Certificate pinning bypass attempts

SSL/TLS Analysis Tools

# SSL/TLS Analysis openssl s_client -connect target:443 openssl x509 -in cert.pem -text -noout nmap --script ssl-enum-ciphers -p 443 target # Certificate Validation # - Certificate transparency logs # - SSL certificate monitoring # - Cipher suite analysis

Security Focus: TCP/UDP protocols, port security, connection analysis

Common Threats

  • • Port scanning/enumeration
  • • TCP sequence number attacks
  • • UDP flooding
  • • Connection hijacking
  • • Covert channels

Investigation Priorities

  • • Port scan detection patterns
  • • Unusual protocol combinations
  • • Connection state anomalies
  • • Bandwidth consumption analysis
  • • TCP flag manipulation

Transport Layer Analysis

Common Threats

  • • IP spoofing
  • • Routing attacks
  • • ICMP tunneling
  • • Network reconnaissance
  • • IP fragmentation attacks

Key Investigation Areas

  • • Source/destination IP reputation
  • • Geographic IP analysis
  • • Routing path anomalies
  • • TTL field analysis
  • • IP fragmentation patterns

Network Investigation Tools

SOC Analyst Investigation Workflows

Top-Down Investigation (Layer 7→1)

  1. Layer 7: Check logs, user behavior, malware
  2. Layer 6: Validate certificates, encryption
  3. Layer 5: Review authentication, sessions
  4. Layer 4: Analyze ports, connections
  5. Layer 3: Investigate IP routing, reputation
  6. Layer 2: Check MAC addresses, ARP
  7. Layer 1: Verify device authorization

Bottom-Up Investigation (Layer 1→7)

  1. Layer 1: Confirm device legitimacy
  2. Layer 2: Validate MAC/ARP tables
  3. Layer 3: Trace IP routing path
  4. Layer 4: Analyze connection patterns
  5. Layer 5: Review session management
  6. Layer 6: Check encryption status
  7. Layer 7: Investigate app behavior

Practical Incident Mapping Examples

Lateral Movement Detection

  • Layer 7: Unusual application access patterns
  • Layer 4: New internal TCP connections
  • Layer 3: Internal IP reconnaissance
  • Layer 2: ARP requests for multiple hosts

Data Exfiltration

  • Layer 7: Large file uploads to cloud services
  • Layer 4: High-volume outbound connections
  • Layer 3: Traffic to suspicious external IPs
  • Layer 2: Unusual MAC-to-IP associations

APT Persistence

  • Layer 7: Backdoor application installation
  • Layer 4: Persistent outbound beacons
  • Layer 3: DNS tunneling traffic
  • Layer 2: Rogue device connections
